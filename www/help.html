Overview

PerSE is a visual analytics web application for periodicity detection and analysis.

What is PerSE?

PerSE is a web application designed and built to aid users in the detection and analysis of periodicity within event data sets. Hence, PerSE stands for "Periodicity in Spatio-temporal Events".

The main idea behind PerSE is that different groups of people use different calendars systems, for instance, Gregorian, Islamic (Hijri), Hebrew, and Persian. Therefore, we should analyze event data sets (e.g. car-crashs, crimes, terrorist acts, etc.) through the lens of the calendar system relative to the communities and areas of interest.

PerSE is composed of several crossfiltering views: the Voronoi Subplots Map, Main Toolbar, Attribute View, Time-Wheel, Timeline, and Table. To detect and analyze periodicity within event data sets, users must use the mouse to interactively set and release filters on one or more of the views.

For more information on periodicity, calendars, and why the two are important, please check out the About page.

Voronoi Subplots Map

The Voronoi Subplots Map is the largest view in PerSE. A view is simply a visible top level component of the app. Some people call them widgets, charts, plots, visualizations, etc., but I will refer to them as views. This view gets it's name from being the mix of two different methods: Small Multiples and Voronoi Diagram.

The Voronoi Subplots Map can be used to detect and analyze periodicity at multiple scales through user interaction. The map displays selected events based on their latitude and longitude. You will also notice that events have different colors. These colors are based on the event's Event Type attribute. These colors are also reflected in all the other views. If you click one of the events, you will see a pop-up with the event's description and recorded fatalities.

When you brush the mouse over the map you will notice that semi-transparent white boundary will highlight. This boundary marks one of the a voronoi diagram cells on the map. Furthermore, you will also notice a light gray circle within each boundary. These circles are the voronoi diagram seeds that control the boundaries of the voronoi cells.

When you brush the mouse over a voronoi cell you will also see one of the subplots along the side of the map highlight. The highlighted subplot displays data points located within the boundary of the voronoi cell. These subplots are positioned to be close to their respective seed coordinate.

Subplots within the Voronoi Subplots Map are always plotted with the same scale to prevent misleading analysis, and are linked to other subplots and the map for dynamic interaction. To illustrate this linking, brush the mouse over the arc "January" within one of the subplots. Two things should happen: a) all other subplots will display their "January" value and b) only "January" events will be displayed on the map. Keep in mind the subplots do not perform any filtering beyond the Voronoi Subplots Map.

Main Toolbar

The Main Toolbar is a convenience toolbar used for syncing all views. Its purpose is to change the calendar system used within all views, and/or reset all the filters/views.

When PerSE starts-up all views are synced to one calendar. However, over the course of analysis, you may decide to display one view in Gregorian and another in Islamic. To sync all the views back to one calendar click on the Main Toolbar, and then click your desired calendar. This is essentially the same as changing the calendar in every view manually.

At some point during analysis you may wish to reset all filters/views within PerSE. To reset all filters/views click on the Main Toolbar. This is essentially the same as clicking on every view.

PerSE starts up with the Nigeria data set, but you can view another data set by clicking on the Main Toolbar and select a new data set. The interface should then change over to a the new data within 3 to 6 seconds... It's slow because some of the data sets are pretty large for PerSE.

Attribute View

The Attribute View is a space-saving view that displays all non-required data set attributes (i.e. NOT date or location). The Attribute View initializes with the Event Type displaying, but you can display other non-required attributes by clicking the on the Attribute View toolbar title. There are currently three different non-required attributes that you can view in the Attribute View:

  • Event Type: The type of event. Also used to symbolize all other views.
  • Actor: The acting group or persons in the event; the predicate.
  • Fatalities: Amount of deaths.

The Attribute View handles each non-required attribute in one of two ways: numerical and non-numerical.

Numerical

Numerical attributes like Fatalities are displayed as histograms. You can create a filter on a histogram by pressing and holding the mouse button down with the left mouse button, dragging the mouse across the histogram, and then releasing the mouse button.

Non-numerical

Non-numerical attributes like Event-Type and Actor are displayed as a bar-lots. Events are binned by unique values and symbolized as bars based on count (i.e. frequency) of events. To create a filter on the bar plot click individual bars on the bar plot.

Since each unique value is assigned a rectangle and there could be a large amount of unique values (the Actor attribute has +100 unique values), this plot is scrollable! You can scroll the plot using the mouse wheel or the scroll bar.

Sometimes you may only want to view a single unique value, for instance "Boko Haram" Actor events. You can save your self a lot of time by clicking to clear all events, and then select the desired unique attribute value (see image below).

Time-Wheel

The Time-Wheel is a compact radial plot for displaying units of a calendar system. There are three rings on the Time-Wheel:

  • Outer: Aggregates all selected events by day of month.
  • Middle: Aggregates all selected events month of year.
  • Inner: Aggregates all selected events by day of week.

When an arc is hovered, labels are displayed in the top right corner of the view to provide information on the calendar unit and the frequency (i.e. count) of events.

You can create a filter with this view by clicking any of the arcs. To remove the filter click .

To change the calendar represented click in the view's toolbar, and select a calendar.

Timeline

The Timeline displays the count (i.e. frequency) of events per calendar unit in chronological order in the form of a bar chart. The Timeline's x-axis spans from the floor date to ceiling date of the data set (i.e. oldest to youngest).

Change the calendar unit used to aggregate events in . Potential calendar units include:

  • Year: Aggreagte events by the calendar year.
  • Month: Aggregate events by the calendar month.
  • Week: Aggregate events by the calendar week. (Default)
  • Day: Aggregate events by the calendar day.

You can create a range filter with this view by pressing the left mouse button down and holding it, dragging the mouse across the Timeline, and then releasing the left mouse button.

Much like all the other views, the calendar can be changed with , and the filter can be removed by clicking .

Table

Below the Timeline you will see a table filled with events. This table displays selected events much like all the other views, however it doesn't provide any filtering capability. It's only duty is to display the raw data in chronological order.

The table only shows 25 events at a time. To see the next 25 selected events click . To see the previous selected events click .

You can also click to change the calendar in which the date is displayed.

Although primitive, the Table can be fairly useful. Next time you find an interesting pattern or finding, drill-down on it with all the views and then look at the table to connect the dots.

Views

An in depth description of PerSE's views.

Before we start, what is a view? A view is simply a visible component of the app. Some people call them widgets, charts, plots, visualizations, etc, but we will refer to them as views. PerSE has a total of six top level views:

  1. Attribute View
  2. Main Toolbar
  3. Table
  4. Timeline
  5. Time-Wheel
  6. Voronoi Subplots Map

In the following sections I'll go over the above top level views in detail. If you are brand new to PerSE or are looking for a brief introduction I recommend the Quickstart section on this page.

Attribute View

A space saving view that displays non-required attributes.

The Attribute View was built to display data attributes that PerSE does not require or anticipate. Therefore, the attribute view could potential consist of any amount non-require data attributes. Each non-required data set attribute is displayed as a single plot or chart, and can filter the data set independent of any non-required or required attribute. The content of the plots are colored by the Event Type non-required attribute. The built in data set has three non-required data attributes.

  • Event Type: The type of event recorded.
  • Actor: The acting group or persons in the event; the predicate.
  • Fatalities: Amount of deaths.

Since PerSE is crunched for screen space, only one non-required attribute in the Attribute View is visible at a time. To change the non-required attribute in the Attribute View click the on the Attribute View's toolbar title. You will then be able to select one of the three non-required data attributes.

After selecting a new non-required attribute the Attribute View will change to reflect the newly selected non-required attribute.

Although there are many different types of data (i.e. nominal, ordinal, etc.), the Attribute View currently handles an attribute in one of two ways: numerical and non-numerical. Both are explained in the following paragraphs.

Numerical

If all attribute values are numerical, then an attribute is determined to be a numerical attribute, and will be displayed as a histogram. To illustrate a numerical non-required attribute click the on the Attribute View's toolbar title and select Fatalities. The view will display a histogram of the Fatalities attribute.

Numerical attribute histograms are binned automatically by an internal algorithm.

Non-Numerical

If not all attribute values are numerical, then the attribute is determined to be a Non-numerical attribute, and will be displayed as a bar chart. Events are binned by unique values and symbolized as horizontal rectangles based on count (i.e. frequency) of events.

Since each unique value is assigned a rectangle and there could be a large amount of unique values (the Actor attribute has +100 unique values), this plot is scrollable! You can scroll the plot using the mouse wheel or the scroll bar.

Filter

Numerical

Numerical attributes can perform range filtering on the data set through user interaction. For example, select all events with reported Fatalities of 10 to 50.

To filter a numerical attribute hover the mouse over the histogram, press the left mouse button down and hold it, drag the mouse across the histogram, and then release the mouse. You should see a gray box during this procedure. After you release the mouse, the gray box will disappear and the plot will scale to the selected range.

There are two ways to remove a filter on the histogram. Like all the other views you can remove the active filter by clicking on the view's tool-bar, or you can single click anywhere on the histogram.

Non-Numerical

Non-numerical attributes can perform selection filtering the data set through user interaction. For example, For example, select all events where Actor is "Boko Haram".

To filter a non-numerical attribute single click individual bars with the left mouse button.

As mentioned above, sometimes there can be a large amount of unique values for an attribute. In these situations, it is difficult to drill-down on a specific attribute value. For instance, to select only "Boko Haram" Actor events we would be required to manually deselect +100 rectangles. To bypass this, click to clear/deselect all unique values in the bar plot, and then select the attribute values you are interested in. For instance, to select only "Boko Haram" Actor events click to clear all events in the plot, and then click "Boko Haram" to view only "Boko Haram" events.

Table

Primitive, but surprisingly useful...

The Table is located below the Timeline and can be used to provide context regarding currently selected events (i.e. events that pass every filtering view). Events are displayed as rows, 25 rows displayed at a time, and each row is given a background color to indicate their attribute value (i.e. in the current data set this attribute is Event Type). Furthermore, the table always sorts data in chronological order.

The Table is most useful for verification or examination of findings. To verify an interesting pattern or finding, do your best to drill-down on it with all the top level views and then look at the table to connect the dots.

Future efforts will be made to make this table more like a shoebox so that users can store and export events/data points.

Paging

The Table limits row output to 25 rows at a time. Row output is limited because adding and removing many rows becomes an expensive DOM operation. Especially when the selection is large and changes a lot! To side step this issue the Table implements paging, click in the view's toolbar to see the next 25 events, or click in the view's toolbar to see the previous 25 events.

Change Calendar

Voronoi Subplots Map

A map backed by an adjustable voronoi diagram + small multiples.

The Voronoi Subplots Map is the main view within PerSE. This view is used to detect where and to what extent periodicity exists. The Voronoi Subplots Map combines two well know information visualization methods: the voronoi diagram and small multiples.

In the following sections I will go over the voronoi diagram, small multiples, and how those two concepts integrate into the Voronoi Subplots Map.

Voronoi Diagram

A voronoi diagram is a diagram that partitions a surface based on a set of seeds or points and their nearness/distance to each other. Each seed forms a cell where everything within is closer to its respective seed than to any other seed. Cell boundaries indicate the location where it is equal distance to neighboring seeds.

The Voronoi Subplots in PerSE uses a voronoi diagram to determine which events go to which subplot. This can be useful in situations where boundaries are fuzzy or unknown. For instance, the boundaries between two communities, cities, or cultural groups.

Small Multiples

Small Multiples are a set of similar charts, plots, or gylphs at the same scale. Small Multiples are primarily used for identifying phenomena by comparison. Are some of the multiples similar, different, any outliers? For more information on this method check out Edward Tufte's Envisioning Information (1990).

The Voronoi Subplots Map uses the small multiples as a way to visualize the content within the voronoi diagram cells. The underlying voronoi diagram is used to aggregate events based on their location. Later, these collections of events are visualized using multiples that support reasoning about periodicity. In order link the multiples back to the map, the Voronoi Subplots Map places each small multiple near its respective voronoi seed/cell.

Map

The map displays selected events based on their latitude and longitude and colors them based on the Event Type attribute. If you click one of the events, you will see a pop-up with the event's description and recorded fatalities.

As mentioned above, this view makes use of a voronoi diagram. If you brush the mouse over the map you will notice that a semi-transparent white boundary on the map will highlight. These boundaries are voronoi diagram cells. Furthermore, you will also notice a light gray circle within each boundary. These circles are the voronoi diagram seeds that control the boundaries of the voronoi cells.

Voronoi Seeds

When you brush the mouse over a voronoi cell you will also see one of the subplots along the side of the map highlight. The highlighted subplot displays data points located within the boundary of the voronoi cell. These subplots are positioned to be close to their respective seed coordinate.

Remove

PerSE starts-up with six voronoi seeds on the map. Sometimes six can be to many, especially if there are few data points or if we want to compare less than six regions. You can get rid of a voronoi seed by clicking on the view's toolbar, and then select Remove. Afterward you should see the seeds highlighted. To remove a seed, simply single click the seed with the mouse. If you successfully removed a seed, the seed and it's respective subplot will disappear, and finally the Voronoi Subplots Map will recalculate the voronoi diagram.

After you are finished removing a voronoi seeds, ensure that you disable the Remove setting in by single clicking the Remove setting.

Add

If you have less than six voronoi seeds on the map, then you can add a voronoi seed to the Voronoi Subplots Map. First click on the view's toolbar, and then select Add. Afterward your cursor should light up on the map hinting that you can add a voronoi seed. If you click on the map, a voronoi seed and its respective subplot will be placed on the map, and finally the voronoi diagram will recalculate.

After you are finished adding a voronoi seeds, ensure that you disable the Add setting in by single clicking the Add setting.

Positioning

Unless you change the voronoi seeds Positioning setting the seeds will always snap to a gridded position. This is because the Positioning setting in is set to Auto (Default). Auto can be a nice setting because it attempts to figure out the best position for the voronoi seeds every time the selection changes. For example, if we removed a seed from the Voronoi Subplots Map when Positioning is set to Auto, then the voronoi seeds would automatically grid the remaining space (See the image on the right).

If you want more control over the voronoi seeds, you can set the Positioning to Manual by clicking on the view's toolbar, and then selecting Manual under Positioning. Now voronoi seeds will not reposition when the selection changes. For example, if we removed a seed from the Voronoi Subplots Map when Positioning is set to Manual, then the remaining voronoi seeds would stay in the exact same location, and the voronoi cell boundaries would adjust to compensate for a missing seed.

To move the voronoi seed ensure that Positioning is set to Manual, and click on the view's toolbar, and then select Move. This will allow you to move a voronoi seed and its respective subplot anywhere on the map. After you move the seed the voronoi diagram will recalculate and the voronoi seeds respective subplot will reposition to a better docking station if the layout permits.

Reset

At some point during analysis you may want to reset voronoi seed locations. To reset, click on the view's toolbar, and select Reset. By doing so, voronoi seeds will be added to the map until it reaches the maximum seeds (six). Then the seeds and their respective plots will reposition into a gridded layout similar to Auto Positioning. Finally, the subplots will update with the newly aggregated data.

Subplots

As mentioned above, subplots display event data derived from their respective voronoi cell, and colored by the Event Type attribute.

Subplots within the Voronoi Subplots Map are always plotted with the same scale to prevent misleading analysis, and are linked to other subplots and the map for dynamic interaction. To illustrate this linking, a mouse brush over the arc "January" within one of the subplots will cause two things to happen: a) all other subplots will display their "January" value and b) only "January" events will be displayed on the map. Keep in mind the subplots do not perform any filtering beyond the Voronoi Subplots Map.

There are two types of subplots in the Voronoi Subplots Map: Time-Wheel (default) and Line Plot. To change the Plot Type click on the view's toolbar, and select either Time-Wheel or Line Plot.

Time-Wheel

The Time-Wheel is radial plot used to display calendar units. Each ring around the plot is segmented by arcs which represent a calendar unit. For example, Day of the Month, Month of the Year, Day of the Week. Each ring aggregates all selected events by their date.

The Time-Wheels in the Voronoi Subplots Map are almost exactly the same as the one in the top level view, however there are some small differences. The Time-Wheels in the Voronoi Subplots Map cannot filter outside of the Voronoi Subplots Map, meaning they are not "clickable". Second, since these plots are all linked, they place a burden on the DOMs rendering capabilities. Thus, they do not transition as smoothly as the top level view Time-Wheel.

Line-Plot

The Line Plot is useful for visualizing the linear aspects cyclical units of time. Each line is a temporal segment (e.g. year, month, week) and is segmented by a calendar unit (e.g. month of year, day of month, day of week). Lines are given a Y-value based on the count (i.e. frequency) of events that took place during the temporal segment.

To change the cycle length for all Line Plots within the Voronoi Subplots Map, click clicking in the view's toolbar, and then select an option within the Cycle section.

Change Calendar

Like most of the top level views in PerSE, the Voronoi Subplots Map allows you to change the calendar used to aggregate events. To change the calendar click on the view's toolbar and select one of the calendar systems. After you have changed the calendar, the subplots in the Voronoi Subplots Map will re-aggregate the data with the new calendar.

Filter

Draw

In addition to periodicity detection and analysis, the Voronoi Subplots Map can also be used to perform spatial filters on the entire data set. For example, select all events within a polygon. To create a spatial filter first click on the view's toolbar, and then click Draw. Much like adding a voronoi seed, your cursor will highlight on the map, and you will be able to draw a polygon by clicking to plot a vertex. To end a drawing session double click or single click the starting vertex. After a polygon is created PerSE will filter the entire data set based on the boundaries of the newly created polygon. Note, at this time PerSE only supports one filtering polygon.

Modify

If you would like to change the boundary of a spatial filter polygon click on the view's toolbar, and then click Modify. You will notice that the spatial filter polygon becomes highlighted, and your mouse cursor will snap to the boundary when near. To modify the spatial filter polygon drag a boundary vertex or edge to the desired location. You can also remove a vertex by single clicking the vertex.

Move

If you would like to move the spatial filter polygon (i.e. change coordinates, but preserve the geometry and orientation.), click on the view's toolbar, and click Move. Much like Modify you will notice that the spatial filter polygon becomes highlighted and when you hover over it your cursor will turn into "move-arrows". To move the spatial filter polygon hover over the spatial filter polygon, press the left mouse button down and hold, drag it to the desired location, and release. When you release the mouse button the spatial filter polygon will fix to that location, and the entire data set will update with the new filter.

Remove

There are two ways to remove a spatial filter polygon. The easiest of the two is to simply click on the views toolbar. After you click this button the spatial filter polygon will be removed from the map, and the Voronoi Subplots Map filter will reset.

The long way to remove a spatial filter is to click on the view's toolbar, and then click Remove. This performs the exact same function as the above method, but was included because it naturally fits on the toolbar.

    • Line Plot: Change all subplots to the Line Plot. The Line Plot is useful for visualizing the linear aspects cyclical units of time. Each line is a temporal segment (e.g. year, month, week) and is segmented by a calendar unit (e.g. month of year, day of month, day of week).
  • Cycle: (Line Plot Only) Changes the cycle used to aggregate plot data.
    • Month of Year: Segments data set by the year and bins it by the month. For example, if the data set is 2 years long (Gregorian), then there would be 2 lines.
    • Week of Year: Segments data set by year and bins it by week. For example, if the data set is 2 years long (Gregorian), then there would be 2 lines.
    • Day of Month: Segments data set by month and bins it by day. For example, if the data set is 2 years long (Gregorian), then there would be (12 months * 2 Years) 24 lines.
    • Day of Week: Segments data set by week and bins it by day. For example, if the data set is 2 years long (Gregorian), then there would be (52 weeks * 2 years) 102 lines.
Calendar Change the calendar used to bin events within the subplots.
  • Gregorian: Change the calendar to the Gregorian calendar.
  • Islamic: Change the calendar to the Islamic (Hijri) calendar.
Draw/Edit Draw a spatial filter polygon or edit the spatial filter polygon.
  • Draw: Draw a new spatial filter polygon. Note, PerSE currently supports only one spatial filter polygon at a time.
  • Edit:
    • Modify: Alter the geometry of the spatial filter polygon. Remove a vertex by single clicking. Move a vertex by dragging it with the mouse, and add a vertex by single clicking.
    • Move: Move the entire spatial filter polygon without changing the polygons orientation and geometry by dragging the polygon with the mouse.
  • Remove: Removes the spatial filter polygon. Essentially the same as clicking .
Voronoi Seed Alter the voronoi seeds or change their positioning.
  • Positioning: Change how the voronoi seeds position.
    • Auto: Voronoi seeds automatically position into an evenly spaced grid.
    • Manual: Voronoi seeds fix to a latitude and longitude until manually moved to a different location.
  • Edit: Alter the voronoi seeds.
    • Add: Add a new voronoi seed to the map by single clicking. (Max: 6 seeds, Min: 1 seed)
    • Move: Move a voronoi seed by dragging it with the mouse. Positioning must be set to Manual
    • Remove: Remove a voronoi seed by single clicking the seed.
  • Reset: Resets all voronoi seeds by adding seeds to the map until there are six, then automatically positions the voronoi seeds into an evenly spaced grid.
Filter Remove and spatial filter polygon. This button is enabled if the view is filtering. n/a
