Overview

PerSE is a visual analytics web application to assist in periodicity detection and analysis.

What is PerSE?

PerSE (Periodicity in Spatiotemporal Events) is a web application to assist users in the detect and analysis of periodicity within event data sets. PerSE attempts to tackle detection and analysis through sophisticated aggregation and methods commonly used in the field of Visual Analytics

PerSE was designed and built for human-related event data sets. For instance, car-crash reports, crimes, terrorist acts, etc. Since human-related events are often interrelated with culture, PerSE makes extensive use of the calendar to analyze event data.

For more information on periodicity, calendars, and PerSE, please check out the About page.

Voronoi Subplots Map

The Voronoi Subplots Map is the largest view in PerSE. A view is simply a visible top level component of the app. Some people call them widgets, charts, plots, visualizations, etc, but I will refer to them as a view. This view gets it's name from being the mix of two different methods: Small Multiples and Voronoi Diagram.

The Voronoi Subplots Map can be used to detect and analyze periodicity at multiple scales through user interaction. The map displays selected events based on their latitude and longitude. You will also notice that events have different colors. These colors are currently based on the event's Event Type attribute. These colors are also r reflected in all the other views. You can click events on the map to get a popup with the event's description.

If you brush the mouse over the map you will notice semi-transparent white boundaries on the map will highlight. These are the voronoi diagram cells. Furthermore, you will also notice a light disk-shaped circle within each boundary. These are the voronoi diagram seeds.

Subplots always plotted with the same scale to prevent misleading values. If you hover the mouse over an arc within one of the subplots two things happen. First, the data on the map will be filtered by that attribute. Second, all subplots are dynamically link so that a hover in one is identical to a hover in all other subplots. This means that you can see the actual data point value in all other subplots. Keep in mind the subplots do not perform any filter beyond the Voronoi Subplots Map.

Views

An in depth description of PerSE's views.

Before we start, what is a view? A view is simply a visible component of the app. Some people call them widgets, charts, plots, visualizations, etc, but we will refer to them as views. PerSE has a total of six top level views:

  1. Attribute View
  2. Main Toolbar
  3. Table

In the following sections I'll go over these top level views in detail. If you are brand new to PerSE or are looking for a brief introduction I recommend the Quickstart Guide.

Attribute View

A space saving view that displays non-required attributes.

The Attribute View can consists of 0 or more plots/charts for visualizing non-require data attributes. Each non-required data set attribute can filter the data set independent of any other view or attribute. The built in data set has three non-required data attributes.

  • Event Type: The type of event recorded.
  • Actor: The acting group or persons in the event; the predicate.
  • Fatalities: Amount of deaths.

Since PerSE is crunched for screen space, you can only visualize one attribute in the Attribute View at a time. To change the attribute in the Attribute view click the on the Attribute View's toolbar title. You will then be able to select one of the three non-required data attributes:

After selecting a new attribute the current Attribute View will change to reflect the newly selected attribute.

Although there are many differnt types of data (i.e. nominal, ordinal, etc.), the Attribute currently handles an attribute in one of two ways: numerical and non-numerical. These are explained in the following paragraphs.

Numerical

If all attribute values are numerical, then an attribute is determined to be a numerical attribute, and will be displayed as a histogram. To illustrate this click the on the Attribute Views toolbar title and select Fatalities. The view will display a histogram of the Fatalities attribute.

Histogram binning is automatically determined by an internal algorithm.

Non-Numerical

If not all attribute values are numerical, then the attribute is determined to be a Non-numerical attribute, and will be displayed as a Bar Plot. Events are binned by all possible unique values and symbolized as horizontal rectangles based on count (frequency) of events.

Since each unique value is assigned a rectangle and there could be a large amount of unique values (the Actor attribute has +100 unique values), this plot is scrollable. You can scroll the plot using the mouse wheel or the scroll bar.

Filter

Numerical

Numerical attributes can perform range filtering on the data set through user interaction. For example, select all events with reported Fatalities of 10 to 50.

To filter a Numerical attribute hover the mouse over the histogram, press the left mouse button down and hold it, drag the mouse across the histogram, and then release the mouse. You should see a gray box during this procedure, but after you release the mouse the plot will scale to the selected range.

There are two ways to remove a filter on the histogram. Like all the other views you can remove the active filter by clicking on the toolbar, or you can single click anywhere on the histogram.

Non-Numerical

Non-numerical attributes can filter the data set by selecting or deselecting unique attribute values through user interaction. For example, select all events with reported Actor of "Boko Haram".

The horizontal bar chart can perform attribute filtering on the data set by single clicking individual bars with the left mouse button.

As mentioned above, sometimes there can be a large amount of unique values for an attribute. In these situations, it is difficult to drill down on an attribute value. For instance, to select only "Boko Haram" Actor events would require us to single click +100 rectangles. Therefore, you can click to clear/deselect all unique values for the plot, and then select the few values you are interested in. For instance, now to select only "Boko Haram" Actor events you can click to clear all events in the plot, and then click "Boko Haram" to view only "Boko Haram" events.

Voronoi Subplots Map

A map + subplots backed by an adjustable voronoi diagram.

The Voronoi Subplots Map is the main view within PerSE. This view is used to detect where and to what extent periodicity exists by combining two well know visualization methods: the voronoi diagram and small multiples.

There are two distinct components within this view: a map and a set of plots that sit on top of the map called "subplots". In the following sections we will go over the voronoi diagram, small multiple, and the map and subplots used in PerSE.

Voronoi Diagram

A voronoi diagram is diagram that shows the neighborhood of points, called seeds. To visualize the neighborhood boundaries are created where it is equal distance from one seed to it's neighbor. Therefore, every point within a polygon is nearest-neighbors to the polygon's seed.

A voronoi diagram is a diagram that partitions a surface based on a set of seeds or points and their nearness/distance to each other. Each seed forms a polygon where everything within is closer its respective seed than to any other seed. Polygon boundaries indicate the location where it is equal distance to neighboring seeds.

The Voronoi Subplots in PerSE uses a voronoi diagram to determine what data points go to which plot. This can be useful in situations where boundaries are fuzzy or unknown such as where periodicity exists.

Small Multiples

Small Multiples are a set of same/similar charts, plots, or gylphs at the same scale, and it primarily used for comparison. For more information on this method check out Edward Tufte's Envisioning Information (1990).

The Voronoi Subplots use the small multiple visualization method with the help of a voronoi diagram. The underlying voronoi diagram is used to aggregate data points based on their location, and these collections of data points are plotted on charts that support reasoning about periodicity. In order link the plots back to the map, PerSE uses positions the plot on the edge of the map near its respective seed on the edge.

Map

The map displays selected events based on their latitude and longitude and colors them ba sed on a specified attribute. In the case of our built in data set on Nigeria, the map colors events by their Event-Type (i.e. Violence against civilians, Battle no change in territory, etc.) If you hover over these events and single-click the left mouse button, you will see a popup that displays a brief description of the event.

As mentioned above, this view makes use of a voronoi diagram. If you brush the mouse over the map you will notice that semi-transparent white boundaries on the map will highlight. These are the voronoi diagram cells. Furthermore, you will also notice a light disk-shaped circle within each boundary. These are the voronoi diagram seeds.

Voronoi Seeds

In addition the voronoi cell boundaries highlighting on mouse brushes, you will also see one of the subplots along the side of the map highlighting. The highlighted subplot displays data points within the boundary of the voronoi cell. These subplots are positioned to be close to their respective seed coordinate.

Remove

PerSE starts-up with six voronoi seeds on the map. Sometimes six can be to many, especially if there are few data points or if we want to compare less the six regions. If this is the case we can get rid of a voronoi seed by clicking on the toolbar, and then select Remove. Afterward you should see the seeds highlighted. If you click one of the seeds the chart will eliminate the seed and its respective subplot, and recalculate the voronoi diagram.

Add

If you have less than six voronoi seeds on the map, then you can add a voronoi seed in a similiar way. First click on the toolbar, and then select Add. Afterward your cursor should light up on the map hinting that you can add a voronoi seed. If you click on the map, within the extent of the data points, a voronoi seed and its respective subplot will be placed on the map, and the voronoi diagram will recalculate.

Positioning

If you have been following the quick start up until now you will have noticed that the voronoi seeds always snap to a gridded position. This is because the Positioning setting in is set to Auto. Auto can be a nice setting because it attempts to figure out the best position for the voronoi seeds every time the selection changes. Therefore, if the only points on the map were in the South-West corner then the voronoi seeds would try to grid the South-West corner.

What if you want more control over the voronoi seeds? Well, you can by changing the positioning to to Manual. To do that click on the toolbar, and then select Manual. Now voronoi seeds will not reposition when the selection changes. If you would like to move the voronoi seed click on the toolbar, and then select Move. This will allow you to move a voronoi seed along with its respective subplot around the map. After you move the seed the voronoi diagram will recalculate and the voronoi seeds respective subplot will reposition to a better docking station if the layout permits.

**Note, Move is only enabled when Positioning is set to Manual.

Reset

At some point during analysis you may want to revert back to the default voronoi seeds. To do this click on the toolbar, and select the Reset. By doing so, voronoi seeds will be added to the map until it reaches the maximum seeds (six). Then the seeds and their respective plots will be repositioning into a gridded layout similar to Auto Positioning . Finally, the subplots will update with the newly aggregated data.

Subplots

As mentioned above the subplots display event data derived from their respective voronoi cell. Each subplot is linked to all other subplots. Meaning, if you hover over an aggregate in one subplot, it will simulate the same hover in all the other subplot. The details for each hover are labeled in the top right corner of the subplot. This type of labeling is mirrored in all the top level views too.

The map is also linked to the subplots. Event points not associated with the hovered aggregate of the subplot will disappear on subplot hovering. Despite the dynamic linking on mouse hovers and brushes, this action has no filtering capabilities outside of the Voronoi Subplots Map.

There are two types of subplots in the Voronoi Subplots Map: Time-Wheel (default) and Line Plot. To change the Plot Type click on the toolbar and select either Time-Wheel or Line Plot.

Time-Wheel

The Time-Wheel is a useful for visualizing cyclical units of time. Each ring around the plot is segmented by arcs which represent a common calendar aggregates. For example, Day of the Month, Month of the Year, Day of the Week. Each ring aggregates all selected events by their date.

The Time-Wheels in the Voronoi Subplots Map are almost exactly the same as the one in the top level view, however there are some small differences. The Time-Wheels in the Voronoi Subplots Map cannot filter outside of the Voronoi Subplots Map, meaning they are not "clickable". Second, since these plots are all linked, they place a burden on the DOMs rendering capabilites. Thus, they do not transition as smoothly as the top level view Time-Wheel.

Line-Plot

The Line Plot is useful for visualizing the linear aspects cyclical units. Each line is a temporal segment (e.g. day, month, week) determined by a cycle length (e.g. month of year, day of week). Lines are given a Y-value based on the count/frequency of events that took place during the temporal segment.

You may change the cycle length for all Line Plots within the Voronoi Subplots Map by clicking in the toolbar, and then selecting and option within the Cycle section.

Change Calendar

Like most of the top level views in PerSE, the Voronoi Subplots Map allows you to change the calendar used to aggregate events. To change the calendar click on the toolbar and select one of the calendar systems. After you have changed the calendar, the subplots in the Voronoi Subplots Map will re-aggregate the data with the new calendar.

Filter

Draw

In addition for periodicity detection and analysis, the Voronoi Subplots Map can also be used to perform spatial filters on the entire data set. To create a spatial filter first click on the toolbar, and then click Draw. Much like adding a voronoi seed, your cursor will highlight on the map, and you will be able to draw a polygon by clicking to plot a vertex. To end a drawing session double click or single click the starting vertex. After a polygon is created PerSE will filter the entire data set based on the boundaries of the newly created polygon. Note, at this time PerSE only supports one filtering polygon.

Modify

If you would like to change the boundary of a spatial filter polygon click on the toolbar, and then click Modify. You will notice that the spatial filter polygon becomes highlighted, and your mouse cursor will snap to the boundary when near. To modify the spatial filter polygon drag a boundary vertex or edge to the desired location. You can also remove a vertex by single clicking the vertex.

Move

If you would like to move the spatial filter polygon (i.e. change coordinates, but preserve the geometry and orientation.), click on the toolbar, and click Move. Much like Modify you will notice that the spatial filter polygon becomes highlighted and when you hover over it your cursor will turn into move-arrows. To move the spatial filter polygon hover over the spatial filter polygon, press-and-hold mouse button down, and drag it to the desired location. When you release the mouse button the spatial filter polygon will fix to that location, and the entire data set will update with the new filter.

Remove

There are two ways to remove a spatial filter polygon. The easiest of the two is to simply click on the toolbar. After you click this button the spatial filter polygon will be removed from the map, and the Vornoi Subplots Map will reset. Also note the button will look like this because there the Voronoi Subplots Map is not filtering the data set. This mechanism is the same on all other top level views in PerSE.

The long way to remove a spatial fitler polygon is to click on the toolbar, and then click Remove. This performs the exact same function as the above method, but was included in this drop-down because it naturally fits there.

    • Move: Move a voronoi seed by dragging it with the mouse. Positioning must be set to Manual
    • Remove: Remove a voronoi seed by single clicking the seed.
  • Reset: Resets all voronoi seeds by adding seeds to the map until there are 6, then automatically positions the voronoi seeds into an evenly spaced grid.
