Views

An in depth description of PerSE's views.

Before we start, what is a view? A view is simply a visible component of the app. Some people call them widgets, charts, plots, visualizations, etc, but we will refer to them as views. PerSE has a total of eight views: Categorical Plot, Line Plot, Numerical Plot, Main Toolbar, Table, Timeline, Time-Wheel, and the Voronoi Map

Voronoi Subplots

A map + subplots backed by an adjustable voronoi diagram.

The Voronoi Subplots is the main view within PerSE. This view is used to detect where and to what extent periodicity exists by combining two well know visualization methods: the voronoi diagram and small multiples.

There are two distinct components within this view: a map and a set of plots that sit on top of the map. In the following sections we will go over what a voronoi diagram is and how it's used in combination with small multiple.

Voronoi Diagram

A voronoi diagram is a diagram that partitions a surface based on a set of seeds or points and their nearness/distance to each other. Each seed forms a polygon where everything within is closer its respective seed than to any other seed. Polygon boundaries indicate the location where it is equal distance to neighboring seeds.

The Voronoi Subplots in PerSE uses a voronoi diagram to determine what data points go to which plot. This can be useful in situations where boundaries are fuzzy or unknown such as where periodicity exists.

Small Multiple

A Small Multiple is a set of same/similar charts, plots, or gylphs at the same scale, and it primarily used for comparison. For more information on this method check out Edward Tufte's Envisioning Information (1990).

The Voronoi Subplots use the small multiple visualization method with the help of a voronoi diagram. The underlying voronoi diagram is used to aggregate data points based on their location, and these collections of data points are plotted on charts that support reasoning about periodicity. In order link the plots back to the map, PerSE uses positions the plot on the edge of the map near its respective seed on the edge.

Table

Primitive, but surprisingly useful...

The table can be used to provide context regarding currently selected events (i.e. events that pass every filtering view). This is useful because it can be used to validate findings. Although the table is fairly primitive the interface would be seriously lacking with-out complete access to the raw data.

Future efforts will be made to make this table more like a shoebox so that users can store and export events/data points.

Features

  1. Gather contextual information by reading about the event in the Description column.
  2. Rows are colored based on the events respective Context Attribute color.
  3. Page left or right to access more events/data points.
  4. Convert date column to a different calendar system.

Buttons

A visual index of buttons used in PerSE.

  • Calendar

  • Description:

    Change the calendar systems used by the plot. If this in in the "All Views" panel then it would change all plots to the selected calendar system.

  • Options:

    • Gregorian: Aggregate events by the Gregorian Calendar.
    • Islamic: Aggregate events by the Islamic Calendar.

  • Clear

  • Description:

    Clears all attributes from the plot. This can be useful for viewing all but one or two attributes without having to deselect all other attributes. For example, if you were only interested in looking a "Boko Haram" attributed events, then you could click the Clear button and then click the "Boko Haram" attribute.

  • Options:

    n/a

  • Chart

  • Description:

    Adjust charting parameters for a given view. For example, you can adjust the aggregation of data in Timeline from Week to Month. As each view has different charting parameters, this may vary between each view.

  • Options:

    • Voronoi Map
      • Plot Type: Changes the type of plots within the Voronoi Plot.
        • Line Plot: A plot that displays the data in the form of a line where each line is an temporal segment determined by a cycle length (i.e. month of year, day of week). This can be useful for visualizing individual, rather than viewing all of them like in the Time-Wheel.
        • Time-Wheel: A plot that displays the data in the form of a wheel. This is useful for displaying the cyclical aspects of temporal data. Data points are binned by their respective temporal aggregate (i.e. Month of Year, Day of Week). As multiple cycles are viewed in one display, cycle settings are irrelevant.
      • Cycle: (Line Plot Only) Changed the cycle used to aggregate plot data.
        • Month of Year: Segments data set by the year and bins it by the month. For example, if the data set is 2 years long (Gregorian), then there would be 2 lines.
        • Week of Year: Segments data set by year and bins it by week. For example, if the data set is 2 years long (Gregorian), then there would be 2 lines.
        • Day of Month: Segments data set by month and bins it by day. For example, if the data set is 2 years long (Gregorian), then there would be (12 months * 2 Years) 24 lines.
        • Day of Week: Segments data set by week and bins it by day. For example, if the data set is 2 years long (Gregorian), then there would be (52 weeks * 2 years) 102 lines.
    • Timeline
      • Cycle: Changes the binning, or temporal resolution, of the timeline. this can be useful to show the differences of calendars at varying temporal scales.
        • Year: Bins the data set by year.
        • Month: Bins the data set by month.
        • Week: Bins the data set by week.
        • Day: Dins the data set by day.

  • Edit

  • Description:

    Edit/Draw the spatial filter polygon. PerSE currently supports only one spatial filter polygon.

  • Options:

    • Draw: Draw a spatial filter polygon. Note, PerSE currently supports only one spatial filter polygon.
    • Edit:
      • Modify: If there is a spatial filter polygon present, then this can be used to modify the boundaries of the polygon. Single-click a vertex to remove it. Drag a vertex reposition the vertex. Single-click when not over a vertext to add a vertext to the geometry.
      • Move: If there is a spatial filter polygon present, then this can be used to move the polygon around the map while preserving the polygon's geometry.
    • Remove: If there is a spatial filter polygon present, then this can be used to remove it. By clicking this button the spatial filter polygon will be removed until a new one is drawn. This is the same as clicking the Filter button.

  • Voronoi Seed

  • Description:

    Change settings and alter the voronoi seed. A voronoi seed is a coordinate that controls how the spatial boundaries defined. These spatial boundaries are used to determine which data points are sent to which voronoi plot (i.e. the plots surrounding the map).

  • Options:

    • Positioning: Change the way voronoi seeds are positioned.
      • Auto: Voronoi seeds are automatically positioned at equal intervals based on the spatial extent of the underlying event data. When a seed is added, seed is removed, or when the selection changes the coordinate will automatically reposition. As this is determined by an algorithm, the option to Move a voronoi coordinate is disabled, unlike on Fixed.
      • Fixed: Voronoi seeds are fixed to their respective coordinates. When a seed is added, seed is removed, or when the selection changes the seeds will not reposition. Therefore, the ability to move any voronoi seed is enabled, unlike on Auto.
    • Edit:
      • Add: Add a voronoi seed to the map. This also adds a voronoi plot to the Voronoi Map. (Max: 6 seeds, Min: 1 seed)
      • Move: Move a voronoi seed on the map. This also repositions the corresponding voronoi plot to. Note, Positioning must me set to Fixed.
      • Remove: Remove a voronoi seed from the map. This also removes the corresponding voronoi plot from the Voronoi Map. (Max: 6 seeds, Min: 1 seed)
    • Reset: Resets the voronoi seeds. Adds voronoi seeds until there are 6, and then repositions them at equal intervals based on the spatial extent of the underlying event data.
